# Code of Conduct

## Purpose

CT Toolkit is an open-source security project. We want the community to be technically rigorous, respectful, and safe for contributors, users, and researchers.

This Code of Conduct applies to repository discussions, issues, pull requests, documentation, and other project-managed spaces.

## Expected Behavior

Participants are expected to:

- Be respectful, professional, and constructive.
- Focus feedback on ideas, code, documentation, and behavior rather than individuals.
- Share technical disagreement clearly and with evidence.
- Assume good intent unless there is clear reason not to.
- Help keep discussions relevant to the project and useful to other contributors.
- Respect maintainer decisions about scope, security, and project direction.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory or hateful language.
- Trolling, deliberate disruption, or bad-faith engagement.
- Publishing private information without explicit permission.
- Spamming, off-topic promotion, or repeated low-signal issue filing.
- Pressuring maintainers for support outside the channels documented in [SUPPORT.md](SUPPORT.md).
- Public disclosure of security vulnerabilities before coordinated remediation. See [SECURITY.md](SECURITY.md).

## Enforcement

Project maintainers may edit, hide, lock, or remove content that violates this Code of Conduct. They may also warn, temporarily restrict, or permanently ban contributors from project spaces depending on severity and pattern of behavior.

Maintainers will aim to enforce this policy consistently and in proportion to the situation.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, report it privately to the project maintainer.

When reporting, include:

- A description of what happened.
- Links, screenshots, or other relevant context.
- Any immediate safety or moderation concerns.

Security issues should not be reported through public issues. Follow the process in [SECURITY.md](SECURITY.md).

## Scope and Project Standards

This project values direct technical discussion. Strong disagreement is acceptable; disrespect is not. Contributors should expect code review to be rigorous, especially for security-sensitive changes, but never abusive.
