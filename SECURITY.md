# Security Policy

## Supported Versions

Currently, only the latest release of the Computational Theseus Toolkit (CT Toolkit) receives security updates. If you are using an older version, please upgrade to the latest stable release to ensure you have the most up-to-date security patches.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.3.x | :white_check_mark: |
| < 0.3.0  | :x:                |

## Reporting a Vulnerability

Your security is very important to us. If you discover a security vulnerability within the CT Toolkit, please follow these guidelines:

1. **Do NOT open a public issue.** Publicly disclosing a vulnerability can put users at risk before a patch is available.
2. **Contact the maintainer directly.** Please report security issues by emailing the project maintainer directly (Hakan Damar) or through a private vulnerability report if enabled on GitHub.
3. **Provide context.** In your report, please include:
   - A detailed description of the vulnerability.
   - Steps to reproduce the issue.
   - Any potential impact passing through identity continuity constraints or provenance logs.

### Response Timeline

We will acknowledge receipt of your vulnerability report within 48 hours. We strive to provide a patch or mitigation strategy as quickly as possible, usually within 7 days of confirmation.

Thank you for helping keep the CT Toolkit and the community safe!
