# Support

## Getting Help

Use the following channels depending on what you need:

### Questions and usage help

Open a GitHub issue if you need help using CT Toolkit, want clarification on configuration, or are unsure how to integrate a feature into your stack.

To get useful help quickly, include:

- CT Toolkit version
- Python version
- Provider and model details
- Minimal code snippet or command
- Error message and traceback

### Bug reports

Open a GitHub issue for reproducible bugs.

Please include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment details
- Relevant logs or screenshots

### Feature requests

Open a GitHub issue describing:

- The problem you are trying to solve
- Why the current behavior is insufficient
- The API or workflow you would like to see

### Security issues

Do not report security vulnerabilities publicly.

Follow the private disclosure process described in [SECURITY.md](SECURITY.md).

## Response Expectations

This is an open-source project and response times may vary. Maintainers will prioritize:

- Security issues
- Reproducible bugs
- Documentation gaps that block adoption
- High-signal feature requests aligned with the project roadmap

## Before Opening an Issue

Please check these resources first:

- [README.md](README.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- [SECURITY.md](SECURITY.md)
- [website/docs](website/docs)

Well-scoped issues with reproduction steps are much easier to resolve.
